# 조선시대 인물 성향 테스트 결과 페이지 고도화 프롬프트

## 1. 작업 목적

현재 구현된 **「당신이 조선시대에 태어났다면?」 성향 테스트**의 결과 페이지를 개선해 주세요.

현재 결과 페이지는 다음 정보를 표시하고 있습니다.

- 최종 매칭 인물 또는 직업
- 한 줄 소개
- 대표 성향 3개
- 강점 3개
- 8개 성향 점수 그래프
- 현대식 조언

기존 결과 정보는 유지하되, 단순히 인물 이름과 그래프만 보여주는 방식에서 벗어나 사용자가 자신의 성격과 행동 특성을 더 풍부하게 이해할 수 있도록 결과 설명을 확장해 주세요.

결과 화면은 일반적인 MBTI 결과 페이지처럼 읽는 재미가 있어야 하지만, MBTI 유형을 직접 추론하거나 사용자가 특정 MBTI라고 단정해서는 안 됩니다.

이 테스트의 결과는 아래 8개 성향 점수와 조선시대 인물 프로필을 기반으로 생성해야 합니다.

---

## 2. 반드시 지켜야 할 원칙

### 2.1 결과 설명의 근거

모든 설명은 반드시 다음 데이터에 근거해 생성해야 합니다.

1. 사용자의 정규화된 8개 성향 점수
2. 점수가 가장 높은 상위 성향 3개
3. 점수가 가장 낮은 하위 성향 1~2개
4. 상위 성향 두 개의 조합
5. 최종 매칭 인물의 고정 프로필

결과 문장을 무작위로 생성하거나, 점수와 관계없는 성격 설명을 출력하지 마세요.

### 2.2 심리검사처럼 과장하지 않기

이 테스트는 임상적·전문적 심리검사가 아니라 오락 및 자기 탐색을 위한 성향 테스트입니다.

따라서 다음과 같은 표현은 사용하지 마세요.

- 검증된 심리 진단 결과입니다.
- 당신의 실제 성격은 반드시 이렇습니다.
- 당신은 특정 MBTI입니다.
- 당신에게 정신적 또는 심리적 문제가 있습니다.
- 이 결과는 과학적으로 확정된 성격입니다.

대신 다음과 같이 표현하세요.

- 답변 선택을 바탕으로 보면 이러한 경향이 나타납니다.
- 이런 상황에서 이와 같은 선택을 선호할 가능성이 있습니다.
- 현재 응답에서는 이러한 성향이 상대적으로 높게 나타났습니다.
- 오락과 자기 탐색을 위한 결과입니다.

결과 페이지 하단에는 반드시 다음 안내 문구를 표시하세요.

> 본 결과는 사용자의 선택을 바탕으로 구성된 오락 및 자기 탐색용 콘텐츠이며, 전문적인 심리검사나 진단을 목적으로 하지 않습니다.

### 2.3 인물에 대한 역사적 단정 금지

결과에 사용되는 조선시대 인물과 직업은 사용자의 성향을 재미있게 표현하기 위한 유형입니다.

역사 인물의 실제 성격을 심리학적으로 진단한 것처럼 표현하지 마세요.

## 3. 성향 데이터

테스트에서 사용하는 성향은 총 8개입니다.

```javascript
const TRAITS = [
  "curiosity",
  "justice",
  "adventure",
  "creativity",
  "empathy",
  "practicality",
  "power",
  "honor"
];
```

사용자 화면에서는 다음 이름으로 표시합니다.

```javascript
const TRAIT_LABELS = {
  curiosity: "탐구심",
  justice: "정의감",
  adventure: "모험심",
  creativity: "창의성",
  empathy: "공감 능력",
  practicality: "현실 감각",
  power: "영향력",
  honor: "명예 지향"
};
```

내부 키는 `power`를 그대로 사용하되, 사용자 화면에서는 부정적으로 보일 수 있는 `권력욕` 대신 `영향력` 또는 `주도성`으로 표현하세요.

## 4. 점수 단계 판정

각 성향 점수는 0점부터 10점 사이의 정규화된 값입니다.

점수를 다음 네 단계로 구분하세요.

```javascript
export function getTraitLevel(score) {
  if (score >= 7.5) return "veryHigh";
  if (score >= 6.0) return "high";
  if (score >= 4.0) return "middle";
  return "low";
}
```

단계 의미는 다음과 같습니다.

| 단계 | 점수 | 의미 |
|---|---:|---|
| veryHigh | 7.5 이상 | 결과를 강하게 특징짓는 핵심 성향 |
| high | 6.0 이상 7.5 미만 | 비교적 뚜렷하게 나타난 성향 |
| middle | 4.0 이상 6.0 미만 | 상황에 따라 유연하게 나타나는 성향 |
| low | 4.0 미만 | 상대적으로 우선순위가 낮은 성향 |

`low`는 결함이나 부족함을 의미하지 않습니다.  
낮은 점수는 해당 성향보다 반대편의 가치나 다른 판단 기준을 더 선호한다는 뜻으로 설명하세요.

예를 들어 모험심 점수가 낮다고 해서 사용자를 겁이 많다고 표현하지 말고 다음처럼 표현해야 합니다.

> 불확실성이 큰 선택보다 충분히 검토된 안정적인 방식을 선호하는 편입니다.

## 6. 상위 성향 조합 분석

두 번째 이미지처럼 풍부한 결과를 만들기 위해 개별 성향만 설명하지 말고, 사용자의 상위 성향 두 개를 조합한 해석을 출력하세요.

상위 성향은 점수가 높은 순서로 정렬하여 가져옵니다.

```javascript
export function getTopTraits(scores, count = 3) {
  return Object.entries(scores)
    .sort((a, b) => b[1] - a[1])
    .slice(0, count)
    .map(([trait, score]) => ({ trait, score }));
}
```

조합 키는 성향 이름을 알파벳순으로 정렬하거나, 양방향 키를 모두 처리하여 순서에 영향을 받지 않게 구현하세요.

권장 파일:

```text
src/data/traitCombinations.js
```

## 7. 낮은 성향 해석

결과에는 상위 성향뿐 아니라 가장 낮은 성향 하나도 활용하세요.

단, 부정적인 평가가 아니라 사용자가 상대적으로 덜 우선하는 가치로 설명해야 합니다.

예시:

```javascript
export const lowTraitDescriptions = {
  curiosity:

  justice:
```

## 8. 결과 설명 생성 규칙

결과 설명은 LLM이 자유롭게 생성하는 방식이 아니라 정해진 데이터와 규칙을 조합하는 방식으로 구현하세요.

권장 파일:

```text
src/utils/generatePersonalityResult.js
```

반드시 다음 정보를 반환하도록 구현하세요.

```javascript
{
  topTraits: [],
  lowTraits: [],
  combinationDescription: "",
  personalitySummary: [],
  strengths: [],
  caution: "",
  workStyle: "",
  relationshipStyle: "",
  decisionStyle: "",
  modernAdvice: ""
}
```

### 8.1 기본 생성 흐름

```javascript
export function generatePersonalityResult({
  scores,
  matchedFigure
}) {
  const sortedTraits = Object.entries(scores)
    .sort((a, b) => b[1] - a[1]);

  const topTraits = sortedTraits.slice(0, 3);
  const lowTraits = sortedTraits.slice(-2).reverse();

  // 1. 상위 두 성향의 조합 설명
  // 2. 상위 세 성향의 단계별 문장
  // 3. 가장 낮은 성향의 긍정적 해석
  // 4. 매칭 인물 고정 설명
  // 5. 중복 문장 제거
  // 6. 최종 결과 객체 반환
}
```

### 8.2 결과 요약 문장

`personalitySummary`는 4~6개의 문장으로 구성하세요.

권장 순서:

1. 상위 두 성향의 조합 문장
2. 첫 번째 상위 성향 문장
3. 두 번째 상위 성향 문장
4. 세 번째 상위 성향 문장
5. 가장 낮은 성향을 긍정적으로 해석한 문장
6. 매칭 인물과 연결되는 문장

같은 의미가 반복되는 문장은 제거하세요.

### 8.3 문장 작성 방식

다음 표현을 주로 사용하세요.

- ~하는 편입니다.
- ~을 중요하게 생각합니다.
- ~할 가능성이 높습니다.
- ~한 상황에서 강점을 보입니다.
- 현재 응답에서는 ~한 경향이 나타났습니다.

다음처럼 과도하게 단정하는 표현은 피하세요.

- 무조건 ~합니다.
- 절대로 ~하지 않습니다.
- 본질적으로 ~한 사람입니다.
- 항상 ~합니다.
- 반드시 ~하게 됩니다.

## 11. 행동 영역별 성향 추론

두 번째 참고 이미지처럼 결과를 읽는 재미를 높이기 위해 다음 항목을 추가하세요.

### 11.1 일하는 방식

제목:

```text
일할 때의 당신
```

다음 성향을 중심으로 생성합니다.

- 현실 감각
- 창의성
- 영향력
- 탐구심

예시 규칙:

```javascript
function generateWorkStyle(scores) {
  if (scores.practicality >= 7 && scores.power >= 6) {
    return "목표와 현실적인 조건을 빠르게 정리하고, 필요한 역할을 나누어 실제 결과를 만드는 방식에 강합니다.";
  }

  if (scores.creativity >= 7 && scores.curiosity >= 6) {
    return "정해진 절차만 따르기보다 문제의 원인을 탐색하고 새로운 해결책을 시험하는 방식에 잘 맞습니다.";
  }

  if (scores.empathy >= 7 && scores.practicality >= 6) {
    return "구성원들의 상황을 살피면서 모두가 실행할 수 있는 방법으로 업무를 조율하는 편입니다.";
  }

  return "상황에 필요한 역할을 파악하고 자신의 강점을 활용해 안정적으로 일을 진행하는 편입니다.";
}
```

### 11.2 사람들과의 관계

제목:

```text
사람들과 있을 때
```

다음 성향을 중심으로 생성합니다.

- 영향력
- 정의감

## 13. 결과 화면 구성

현재 한 화면에 모든 내용을 길게 나열하기보다 읽기 쉬운 카드 구역으로 분리하세요.

권장 순서:

### 13.1 최종 매칭 영역

```text
최종 매칭 결과

당신은 조선의 목민관형 지방관

사람들의 삶을 가까이 살피고
현실적인 행정으로 바꾸는 책임자형입니다.
```

포함 요소:

- 인물 이미지
- 역할
- 결과 제목
- 한 줄 소개
- 대표 성향 태그 3개

### 13.2 성격 요약 카드

제목:

```text
답변으로 살펴본 당신의 성향
```

내용:

- 4~6개의 개인화된 문장
- 상위 성향 조합 문장을 첫 문장으로 사용
- 문장 사이에 충분한 여백 적용
- 현재처럼 짧은 불릿만 보여주지 말고 자연스러운 설명형 문장 사용

### 13.3 행동 영역 카드

다음 세 항목을 각각 구분해 표시하세요.

```text
일할 때의 당신
사람들과 있을 때
선택의 순간에는
```

각 항목은 1~2문장으로 구성하세요.

### 13.4 강점 카드

제목:

```text
이런 점이 강점이에요
```

개인화된 강점 3~4개를 표시하세요.

### 13.5 주의할 점 카드

제목:

```text
이럴 때는 한 번 더 살펴보세요
```

사용자의 가장 높은 성향을 기반으로 한 주의 문장 1개를 표시하세요.

### 13.6 성향 그래프

기존 8개 막대그래프는 유지하세요.

요구사항:

- 그래프 값은 실제 정규화 점수와 일치해야 합니다.
- 점수가 높은 순서로 정렬하지 말고 기존 성향 순서를 유지해도 됩니다.
- 막대 끝에 점수를 숫자로 표시하세요.
- 대표 성향 3개는 막대 또는 라벨을 조금 더 강조하세요.
- 0점부터 10점까지의 기준선을 명확하게 표시하세요.
- 그래프만 보고 성격을 단정하지 않도록 간단한 안내 문구를 추가하세요.

예시:

> 점수는 좋고 나쁨이 아니라, 이번 답변에서 각 판단 기준을 얼마나 우선했는지를 보여줍니다.

### 13.7 현대식 조언

기존 `현대식 조언`은 유지하되 결과 인물의 고정 문장과 사용자의 상위 성향을 함께 반영하세요.

예시:

> 책임감과 현실 감각이 강한 만큼 일을 혼자 떠안기 쉽습니다. 반복되는 업무는 절차와 시스템으로 정리하고, 다른 사람에게 역할을 나누는 연습을 해보세요.

### 13.8 결과 하단 안내

```text
본 결과는 사용자의 선택을 바탕으로 구성된 오락 및 자기 탐색용 콘텐츠이며,
전문적인 심리검사나 진단을 목적으로 하지 않습니다.
```

## 14. 디자인 방향

현재의 조선시대 한지 시험지 콘셉트는 유지하세요.

다만 결과 내용이 풍부해지면서 한 화면이 지나치게 길고 답답해지지 않도록 다음을 반영하세요.

- 최종 인물 영역은 가장 크게 강조
- 설명 영역은 밝은 한지색 카드로 분리
- 카드마다 얇은 먹색 또는 황토색 테두리 사용
- 제목에는 붓글씨 계열 글꼴 사용
- 본문에는 가독성 좋은 한글 글꼴 사용
- 대표 성향은 붉은 도장 형태의 태그로 표시
- 본문 글자 크기는 모바일에서도 읽기 쉽게 설정
- 문단의 줄 간격을 충분히 확보
- 결과 영역 사이에는 얇은 점선 또는 전통 문양 구분선 사용
- 장식 요소가 글자를 가리지 않도록 주의
- 인물 이미지는 찌그러지지 않게 `object-fit: contain` 사용

모바일에서는 모든 카드가 한 열로 배치되어야 합니다.

PC에서는 최종 인물 이미지와 제목을 좌우로 배치할 수 있지만, 성격 설명 영역은 본문 너비가 너무 넓어지지 않도록 최대 너비를 제한하세요.

## 17. LLM 사용 여부

결과 문장은 기본적으로 정해진 규칙과 문장 데이터로 생성하세요.

LLM API를 사용하더라도 다음 용도로만 제한하세요.

- 이미 생성된 결과 문장을 자연스럽게 다듬기
- 중복되는 문장을 하나로 합치기
- 동일한 의미를 유지하며 문체를 통일하기

LLM이 점수에 없는 성격을 새로 추론하거나, 사용자의 MBTI·정신 상태·직업 적합성을 임의로 단정하게 만들지 마세요.

LLM 호출에 실패해도 기본 규칙 기반 결과가 정상적으로 표시되어야 합니다.

즉, LLM은 필수 기능이 아니라 선택적인 문장 다듬기 기능이어야 합니다.

## 18. 테스트 요구사항

구현이 끝나면 다음을 확인하세요.

### 기능 테스트

- 8개 점수가 모두 결과 생성에 사용되는가
- 상위 성향 3개가 정확하게 계산되는가
- 가장 낮은 성향이 정확하게 계산되는가
- 성향 조합 키의 순서가 달라도 같은 문장이 나오는가
- 동일한 답변에는 항상 동일한 결과 문장이 나오는가
- 결과 문장과 그래프 점수가 일치하는가
- 점수 데이터가 일부 누락되어도 화면이 깨지지 않는가
- 새로고침 후에도 결과가 유지되는가
- 다시 테스트하기를 누르면 이전 결과가 정상적으로 초기화되는가

### 문장 품질 테스트

- 같은 의미의 문장이 반복되지 않는가
- 낮은 성향을 부정적으로 비난하지 않는가
- 사용자를 과도하게 단정하지 않는가
- 모든 문장이 실제 점수 또는 인물 데이터에 근거하는가
- 인물 이름만 바꾸고 동일한 문장이 반복되는 구조가 아닌가
- 한 문장이 지나치게 길지 않은가

### 화면 테스트

- 모바일 360px 너비에서 글자가 잘리지 않는가
- 인물 이미지가 찌그러지지 않는가
- 결과 카드 사이의 간격이 충분한가
- 긴 문장이 카드 밖으로 넘치지 않는가
- 대표 성향 태그가 작은 화면에서도 자연스럽게 줄바꿈되는가
- 그래프의 라벨과 수치가 겹치지 않는가

## 19. 구현 시 주의사항

- 기존 점수 계산 및 인물 매칭 로직을 불필요하게 변경하지 마세요.
- 기존 결과 페이지에서 정상적으로 동작하는 기능을 삭제하지 마세요.
- 현재 프로젝트의 파일 구조와 기술 스택을 먼저 확인한 후 수정하세요.
- 결과 문장 데이터와 생성 함수를 UI 컴포넌트에서 분리하세요.
- 점수가 같을 경우 기존 배열 순서가 아니라 명확한 동점 처리 기준을 사용하세요.
- 숫자는 소수점 첫째 자리까지만 화면에 표시하세요.
- 결과 문장을 생성할 때 `undefined`, 빈 문자열, 중복 문장이 표시되지 않도록 하세요.
- 데이터 로딩 실패 시 사용자에게 오류 메시지와 다시 시도 버튼을 제공하세요.
- 콘솔 오류와 경고가 발생하지 않도록 확인하세요.
